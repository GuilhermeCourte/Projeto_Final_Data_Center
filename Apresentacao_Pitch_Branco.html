Tópico 03 — Stack Tecnológica

Cada peça tem uma justificativa técnica

ESP32

Edge Device

Dual-core com Wi-Fi nativo e baixo custo — processa a regra crítica localmente sem depender da nuvem.

DHT22

Sensor

Temperatura e umidade em um só componente digital, faixa e precisão adequadas a ambiente de TI.

MQTT

Protocolo

Pub/Sub leve e assíncrono, projetado para IoT e redes instáveis — desacopla dispositivo e dashboard.

Node-RED

Nuvem / UI

Orquestração low-code por fluxos e dashboard pronto para gauge, gráfico e botões de controle.

ArduinoJson

Biblioteca

Serializa a telemetria em JSON válido e enxuto, garantindo integridade das chaves no transporte.

PubSubClient

Biblioteca

Cliente MQTT para o ESP32: publica, assina comandos e reconecta automaticamente ao broker.

Wokwi

Simulação

Simula ESP32 + DHT22 + LED, permitindo demonstração reproduzível sem hardware físico.

HiveMQ

Broker MQTT

Broker público que intermedia as mensagens entre Edge e nuvem com autenticação por credenciais.

Tópico 04 — Inteligência na borda

Edge Computing: decidir sem esperar a nuvem

Acionamento autônomo

Ao detectar T > 28 °C, o ESP32 liga o exaustor de forma imediata, independente de qualquer comando da nuvem.

Edge Continuity

Se a conexão com o Node-RED cair, a proteção continua funcionando localmente — atende ao critério de Autonomia.

Prioridade do crítico

O gatilho automático tem prioridade sobre o comando manual: segurança térmica nunca é sobreposta.

// loop() — executado continuamente no ESP32
float temperatura = dht.readTemperature();

// REGRA DE EDGE COMPUTING (autônoma)
if (temperatura > 28.0) {
  digitalWrite(LED_EXAUSTOR, HIGH); // liga sozinho
  Serial.println("ALERTA: modo automatico prioritario");
}

// Telemetria → nuvem (JSON enxuto)
doc["temperatura"] = temperatura;
doc["umidade"]     = umidade;
doc["exaustor"]    = digitalRead(LED_EXAUSTOR) ? "ON" : "OFF";
client.publish("cloudtech/datacenter/monitor", jsonBuffer);

Tópico 05 — Segurança & Escalabilidade

Seguro hoje, pronto para escalar amanhã

Segurança implementada

Autenticação no broker

Conexão MQTT exige usuário e senha obrigatórios — sem credencial, sem acesso.

Tópicos namespaced

Validação de payload

JSON estruturado com chaves fixas evita comandos e dados malformados.

Reconexão resiliente

Evolução para indústria
